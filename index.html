✅ 완전히 구현 가능한 것들

A 측:

  • 캘린더 모드 (기본) — E-Paper는 전원 없이도 화면 유지. RTC 또는 NTP로 날짜 동기화 후 표시하면 됨
  • 하루 1회 새로고침 (00시) — Deep Sleep + Timer Wake-up으로 24시간마다 기상 → 화면 갱신 → 재수면
  • 배터리 효율 — ESP32-S3 Deep Sleep 시 약 10~25µA. E-Paper는 갱신 시에만 전력 소모. 1500mAh 배터리 기준 수개월~1년 운용 가능
  • 물리 버튼으로 정보 전송 — 버튼 인터럽트로 Deep Sleep에서 깨어나 B에 메시지 전송 가능
  • A↔B 무선 통신 — ESP-NOW 프로토콜 사용 시 WiFi AP 연결 없이 초저전력 P2P 통신 가능

B 측:

  • 온도/빛/PIR 센서 — 일반 GPIO + ADC로 연결, 매우 표준적인 구현
  • 4개 버튼 — GPIO 인터럽트로 처리
  • WiFi로 핸드폰 연동 — ESP32 WebServer 또는 MQTT 등으로 구현 가능
  • 날씨 API — WiFi HTTP 요청으로 OpenWeatherMap 등 호출
  • OpenAI API 연동 — HTTPS 요청으로 API 호출 가능 (ESP32-S3는 TLS 지원)
  • A에 정보 표시 요청 — ESP-NOW 또는 WiFi로 B→A 데이터 전송, A가 화면 갱신

⚠️ 구현 가능하지만 제약사항이 있는 것들

ESP32 칩 종류Bluetooth Classic (A2DP)BLE인터넷 라디오 → BT 스피커

✅ B를 오리지널 ESP32 (WROVER 등)로 사용하면 완벽히 가능

오리지널 ESP32는 WiFi + Bluetooth Classic + BLE를 모두 지원합니다.

1. GxEPD2 라이브러리 (권장)

가장 범용적이고 강력한 방법입니다. Adafruit GFX 기반이라 그래픽 함수가 풍부합니다.

#include "GxEPD2_BW.h"

#define PWR 7
#define BUSY 48
#define RES 47
#define DC 46
#define CS 45

GxEPD2_BW<GxEPD2_420_GYE042A87, GxEPD2_420_GYE042A87::HEIGHT> 
  epd(GxEPD2_420_GYE042A87(CS, DC, RES, BUSY));

void epdPower(int state) { 
  pinMode(PWR, OUTPUT);  
  digitalWrite(PWR, state);  // CrowPanel 전용 - 전원 ON/OFF
}

핵심 포인트: CrowPanel은 GPIO7로 디스플레이 전원을 제어합니다. epdPower(HIGH)로 켜고, 갱신 후 epdPower(LOW)로 꺼도 화면은 그대로 유지됩니다.

3. 전체 동작 흐름 (Deep Sleep 포함)

void setup() {
  esp_sleep_wakeup_cause_t wakeup = esp_sleep_get_wakeup_cause();
  
  epdPower(HIGH);
  epdInit();
  
  switch (wakeup) {
    case ESP_SLEEP_WAKEUP_TIMER:
      syncTimeFromB();
      drawCalendar();
      break;
    case ESP_SLEEP_WAKEUP_EXT0:
      requestDataFromB();
      drawBasedOnMode();
      break;
    default:
      drawCalendar();
      break;
  }
  
  epd.hibernate();
  epdPower(LOW);
  
  esp_sleep_enable_timer_wakeup(timeUntilMidnight());
  esp_sleep_enable_ext0_wakeup(GPIO_NUM_2, LOW);
  esp_deep_sleep_start();
}

4. B에서 데이터 받아서 화면 갱신

typedef struct {
  uint8_t mode;
  char title[64];
  char artist[64];
  uint8_t image[1800]; // 120x120 흑백 비트맵
} DisplayPayload;

void onDataRecv(const uint8_t *mac, const uint8_t *data, int len) {
  DisplayPayload *payload = (DisplayPayload *)data;
  epdPower(HIGH);
  epdInit();
  switch (payload->mode) {
    case MODE_MUSIC:
      drawMusicInfo(payload->title, payload->artist);
      break;
    case MODE_WEATHER:
      drawWeather(payload->title);
      break;
  }
  epd.hibernate();
  epdPower(LOW);
}

Deep Sleep 상태에서는 WiFi/BT 라디오가 완전히 꺼져 있기 때문에, ESP-NOW든 WiFi든 무선 신호로 A를 깨울 수 없습니다.

방법 1: B → A 유선 GPIO 연결 (가장 확실)

B에서 A의 RTC_GPIO 핀 하나에 물리적으로 선을 연결합니다.

B (GPIO 출력) ───── 선 1개 ───── A (RTC_GPIO 입력, ext0 wake-up)

// A 측: Deep Sleep 진입 전

// B 측: A를 깨우고 싶을 때

digitalWrite(WAKE_PIN, LOW);

방법 2: 주기적 Light Sleep + 폴링

A가 짧은 주기(예: 30초~5분)로 깨어나서 ESP-NOW 메시지가 있는지 확인합니다.

5초마다상당히 증가 — 배터리 수명 수주로 단축

방법 3: 하이브리드 (권장)

// A 측 - 여러 wake-up 소스 동시 설정

esp_sleep_enable_ext0_wakeup(GPIO_NUM_2, LOW);          // 버튼

B에서 A로 점퍼 와이어 1개만 연결하면, 무선 통신의 저전력 장점을 유지하면서도 B가 언제든 A를 깨울 수 있습니다.

각 모드 상세

1. Active Mode (활성)

일반 동작 상태. WiFi 전송 시 최대 240mA까지 소모합니다.

2. Modem Sleep (모뎀 슬립)

WiFi/BT 라디오만 끄고 CPU는 계속 동작. A 보드에서는 큰 의미 없음.

3. Light Sleep (라이트 슬립)

CPU가 정지하지만 RAM 내용이 보존됩니다. 깨어나면 잠든 시점부터 코드가 이어서 실행됩니다.

Wake-up 소스: 타이머, GPIO (아무 핀 가능), UART 수신, 터치패드, ext0/ext1

장점: 복귀 빠름(1ms 이내), 프로그램 상태 유지
단점: Deep Sleep보다 전류 10~50배 높음

4. Deep Sleep (딥 슬립) ← A 보드의 메인 모드

CPU와 대부분의 RAM이 완전히 꺼짐. 깨어나면 setup()부터 다시 시작.

Wake-up 소스: 타이머, ext0, ext1, 터치패드, ULP 코프로세서

RTC_DATA_ATTR int bootCount = 0;  // Deep Sleep 간에도 유지됨

5. Hibernation (하이버네이션) — 최소 전력

RTC 메모리와 ULP까지 꺼서 전력을 더 줄임.

Wake-up 소스: 타이머, ext0/ext1만 가능 (ULP, 터치 불가)

esp_sleep_pd_config(ESP_PD_DOMAIN_RTC_SLOW_MEM, ESP_PD_OPTION_OFF);
esp_sleep_pd_config(ESP_PD_DOMAIN_RTC_FAST_MEM, ESP_PD_OPTION_OFF);
esp_sleep_pd_config(ESP_PD_DOMAIN_RTC_PERIPH, ESP_PD_OPTION_OFF);
esp_deep_sleep_start();

enum PowerPreset {
  PRESET_FRAME,       // 액자 모드
  PRESET_SMART,       // 스마트 디스플레이
  PRESET_MUSIC,       // 뮤직 플레이어
  PRESET_INTERACTIVE  // 인터랙티브
};

RTC_DATA_ATTR PowerPreset currentPreset = PRESET_FRAME;

void enterSleep() {
  switch (currentPreset) {
    case PRESET_FRAME:
      esp_sleep_enable_timer_wakeup(24ULL * 3600 * 1000000);
      if (wiredMode) 
        esp_sleep_enable_ext1_wakeup(BIT(WIRE_PIN), ESP_EXT1_WAKEUP_ANY_HIGH);
      esp_sleep_enable_ext0_wakeup(BUTTON_PIN, LOW);
      esp_deep_sleep_start();
      break;

    case PRESET_SMART:
      esp_sleep_enable_timer_wakeup(5ULL * 60 * 1000000);
      if (wiredMode)
        esp_sleep_enable_ext1_wakeup(BIT(WIRE_PIN), ESP_EXT1_WAKEUP_ANY_HIGH);
      esp_sleep_enable_ext0_wakeup(BUTTON_PIN, LOW);
      esp_deep_sleep_start();
      break;

    case PRESET_MUSIC:
      esp_sleep_enable_timer_wakeup(30ULL * 1000000);
      if (wiredMode)
        gpio_wakeup_enable(WIRE_PIN, GPIO_INTR_HIGH_LEVEL);
      esp_sleep_enable_gpio_wakeup();
      esp_light_sleep_start();
      handleWakeup();
      break;

    case PRESET_INTERACTIVE:
      esp_wifi_stop();
      break;
  }
}
