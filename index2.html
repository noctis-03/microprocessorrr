B 보드 — 센서 + 날씨 수집기

전원이 켜진 동안 항상 동작하는 "서버" 역할. 카드를 눌러 펼쳐보세요.

1센서 읽기›
DHT11로 온도·습도, LM393으로 밝기, PIR로 움직임을 감지합니다. 0.5초마다 값을 갱신하며, 움직임은 감지 후 3초간 유지됩니다.
2인터넷 날씨 가져오기›
Open-Meteo API에서 대전(위도 36.35, 경도 127.38) 예보를 받아옵니다. 하루 중 0·4·8·12·16·20시 6개 시점의 기온·습도·날씨코드만 골라 저장하고, 30분마다 자동 갱신합니다.
3OLED 화면 표시›
작은 OLED에 시간·온습도·밝기·움직임 여부를 계속 보여줍니다. A 보드에 데이터를 보내면 "Sent to A!" 메시지가 잠깐 표시됩니다.
4WiFi 간편 설정 (AP 모드)›
저장된 WiFi가 없으면 스스로 DualCore-Setup 접속점을 만듭니다. 휴대폰으로 접속해 웹페이지에서 SSID·비밀번호를 입력하면 저장 후 재부팅해 연결합니다.
★핵심: 요청에 응답하기›
A가 MSG_REQUEST를 보내면, B는 즉시 센서값을 읽고 현재 시각 + 날씨 예보 6칸을 묶어 MSG_SENSOR_DATA 패킷으로 되돌려 보냅니다.

A 보드 — 전자잉크 디스플레이

평소엔 딥슬립으로 배터리를 아끼는 저전력 "클라이언트". 카드를 눌러 펼쳐보세요.

A캘린더 모드›
오늘 날짜를 크게 보여주는 일간 뷰와 한 달 달력 월간 뷰로 나뉩니다. MENU 버튼으로 전환하며, 하단에 현재 온습도가 표시됩니다.
B날씨 모드›
B가 보내준 6개 시점 예보를 시각·아이콘·기온·습도로 표시합니다. 날씨 코드에 따라 해·구름·비·눈·안개·뇌우 아이콘을 직접 그립니다.
C스포티파이 모드›
현재는 레이아웃(틀)만 구현된 상태입니다. 곡 제목, 아티스트, 재생바 자리가 그려져 있습니다.
D정보 모드 (EXIT 버튼)›
부팅 횟수, 통신 채널, 시간 동기화 여부, 마지막 갱신 시각 등 시스템 상태를 보여줍니다. 10초 후 자동으로 캘린더로 돌아갑니다.
ERTC 메모리 (잠들어도 기억)›
날짜·날씨·온습도를 RTC_DATA_ATTR 특수 메모리에 저장해 딥슬립 중에도 데이터가 사라지지 않습니다. 깨어나도 이전 정보를 그대로 기억합니다.
F부분 갱신 vs 전체 갱신›
전자잉크는 부분 갱신을 반복하면 잔상이 남기 때문에, 15번 부분 갱신마다 한 번씩 전체 갱신으로 화면을 깨끗하게 유지합니다.

5개 버튼: MENU EXIT UP DOWN OK

A 보드 동작 한 사이클

각 단계를 눌러 상세 설명을 펼쳐보세요. (깨어남 → 화면 갱신 → 다시 잠듦)

1
깨어남 — 버튼 입력 또는 타이머로 딥슬립에서 기상
자정마다(또는 24시간 후) 자동으로 깨어나거나, 사용자가 버튼을 누르면 즉시 깨어납니다.
2
ESP-NOW 초기화 — 통신 채널을 10번으로 고정
두 보드가 같은 채널에 있어야 서로를 찾을 수 있습니다.
3
B에게 데이터 요청 — requestSensorData() 호출
1초간 응답을 기다리고, 안 오면 최대 5번 빠르게 재시도합니다.
4
응답 수신 & 저장 — 센서값 + 시각 + 날씨를 RTC 메모리에 저장
응답이 없으면 이전에 저장해둔 캐시 데이터를 그대로 사용합니다. 덕분에 멈추지 않습니다.
5
화면 갱신 — 현재 모드에 맞춰 전자잉크 화면을 그림
캘린더 / 날씨 / 스포티파이 / 정보 중 현재 모드의 화면을 출력합니다.
6
버튼 입력 대기 (10초) — 모드 변경 / 새로고침
버튼을 누르면 모드를 바꾸거나 화면을 새로고침하고, 아무 입력이 없으면 다음 단계로 넘어갑니다.
7
다시 잠듦 — 캘린더로 복귀 후 딥슬립 진입
화면을 캘린더로 되돌리고, 좌측 상단에 작은 점(슬립 표시)을 찍은 뒤 다음 자정까지 잠듭니다.
ESP32 · ESP-NOW · Open-Meteo API · E-Paper Display
